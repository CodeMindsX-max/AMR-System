# 📥 BV-BRC Data Download Guide

> Step-by-step instructions for downloading the two CSV files needed by this project.

---

## Overview

This project requires **two CSV files** from the [BV-BRC database](https://www.bv-brc.org) (Bacterial and Viral Bioinformatics Resource Center):

| File | Purpose | Save Location |
|------|---------|---------------|
| `amr_phenotype.csv` | Resistance labels + MIC values | `data/raw/amr_phenotype.csv` |
| `sp_genes.csv` | Gene presence/absence features | `data/raw/sp_genes.csv` |

Both files must cover **Acinetobacter baumannii** (Taxon ID 470).

---

## Method 1 — Manual Browser Download

### Step 1: Download `amr_phenotype.csv`

1. Open: [https://www.bv-brc.org/view/Taxonomy/470#view_tab=amr](https://www.bv-brc.org/view/Taxonomy/470#view_tab=amr)
2. In the table, click the **Filter** icon on the **Antibiotic** column
3. Type `meropenem` and apply the filter
4. Click the **Download** button (↓ icon) → select **CSV**
5. Save the file as `data/raw/amr_phenotype.csv`

> **Expected:** ~10,000+ rows with columns including `Genome ID`, `Antibiotic`, `Resistant Phenotype`, `Measurement Value`

### Step 2: Download `sp_genes.csv`

1. Open: [https://www.bv-brc.org/view/Taxonomy/470#view_tab=specialtyGenes](https://www.bv-brc.org/view/Taxonomy/470#view_tab=specialtyGenes)
2. Filter the **Property** column → select `AMR`
3. Click the **Download** button (↓ icon) → select **CSV**
4. Save the file as `data/raw/sp_genes.csv`

> **⚠️ Important:** BV-BRC browser limits this download to **1,000 rows**. This is the main reason the model has fewer features than published studies. See [Why is accuracy lower?](README.md#-why-is-accuracy-lower-than-literature) in the README.

---

## Method 2 — Automatic Download via CLI

The project includes a built-in downloader that uses the BV-BRC public API:

```bash
python main.py download
```

This bypasses the browser's 1,000-row limit for `sp_genes.csv` (requests up to 500,000 rows via API).

> **Note:** The API download may take 1–2 minutes depending on your internet connection. The script will show progress.

---

## After Downloading

Your `data/` directory should look like this:

```
data/
├── raw/
│   ├── amr_phenotype.csv    ✅
│   └── sp_genes.csv         ✅
└── processed/               (auto-generated by pipeline)
```

Then run the pipeline:

```bash
# CLI
python main.py pipeline

# Streamlit Dashboard
streamlit run streamlit_app/app.py
```

## Troubleshooting

### "No genome overlap between the two CSV files"
Both files must be downloaded for the **same organism** (A. baumannii, Taxon 470). Re-download both from the links above.

### "KeyError: 'antibiotic'"
BV-BRC sometimes uses Title Case column names (`Antibiotic` vs `antibiotic`). The pipeline auto-normalises these, but if you see this error, check the column names in your CSV and add mappings to `_COL_ALIASES` in `controllers/data_controller.py`.

### "9,000+ rows dropped"
This is normal if you don't have the MIC fallback encoding. Make sure you have the latest `data_controller.py` (v3), which recovers MIC-only rows using CLSI breakpoints.

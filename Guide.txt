What is this project? (simple explanation)
Problem: Acinetobacter baumannii is a dangerous hospital bacteria. Testing if it resists Meropenem (a strong antibiotic) takes 24–72 hours in a lab.

Solution: Use machine learning to predict resistance from DNA/gene data in seconds.

Input: Which AMR genes are present in a bacterial genome (1 = present, 0 = absent).

Output: Resistant (1) or Susceptible (0), plus SHAP explanations showing which genes drove the prediction.

How the project flows (big picture)
BV-BRC CSV files OR synthetic data
DataController
Clean MIC values + build gene matrix
Train/Test split + SMOTE balancing
TrainController - 4 ML models
EvalController - metrics + ROC curves
SHAPAnalyser - explain predictions

Train models — controllers/train_controller.py + models/amr_models.py

Explain with SHAP — views/shap_views.py

Best way to understand the project (recommended order)
1. Run the CLI pipeline first
py main.py pipeline
Watch the terminal output — it prints each phase, top genes, metrics, and SHAP results.

2. Open the Streamlit dashboard
py main.py streamlit
Use the sidebar and go through pages in order:

Overview — what the project does
Dataset Explorer — see the data
Preprocessing — how features are built
Model Training — cross-validation results
Evaluation — ROC curves, metrics table
SHAP — which genes matter most
Literature Comparison — vs published papers
Live Prediction — try your own gene combinations
3. Read main.py (only ~60 lines)
This is the “story” of the project — it calls each controller in sequence.

4. Trace one path: data → model → prediction
Data path (data_controller.py):

Load 2 CSVs → normalize column names → parse MIC (>16, <=2) → label Resistant/Susceptible → pivot genes into 0/1 matrix → split 80/20 → SMOTE on training only
Model path (amr_models.py):

4 classifiers trained on balanced data
XGBoost is primary; uses scale_pos_weight for class imbalance
Explainability (shap_views.py):

SHAP answers: “Why did the model predict Resistant for this genome?”
5. Use the Jupyter notebook (for step-by-step exploration)
py notebooks/generate_notebook.py
Then open notebooks/AMR_ML_Project.ipynb in VS Code and run cells one by one.

Quick mental model
Think of it as a factory line:

Raw genomic CSVs
    → Clean & label (Resistant/Susceptible)
    → Gene presence matrix (rows = bacteria, columns = genes)
    → Train 4 ML models
    → Test on held-out 20%
    → Explain with SHAP
    → Show in dashboard
Clinical idea: If genes like blaOXA-23, blaNDM-1, adeB are present, the bacteria is more likely to resist Meropenem — the model learns these patterns from data.
